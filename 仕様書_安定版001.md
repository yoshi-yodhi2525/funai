# ネポン御見積書ウィジェット 仕様書
**安定版001** — 2026-04-09

---

## 1. 概要

Zoho CRM の **Quotes（御見積）モジュール**に埋め込むカスタムウィジェット。  
見積明細の入力・計算・PDF出力・CRM保存を一元管理する。

## 5. 計算仕様

### 5-1. 金額計算
```
金額 = 定価（Unit_Price / field6） × 数量
原価金額 = 原価 × 数量
```

### 5-2. 歩単・歩工
```
歩単 = マスタ固定値（計算しない）
歩工 = 数量 × 歩単
```
- 機器・資材行: 歩単 = 0（工事なし）
- 工事行: 歩単 = 0.5〜2.5（マスタ値）

### 5-3. 減衰計算
歩単 > 0 のアイテムを **calcCategory** でグループ化し、グループごとに減衰適用。

```
d    = Σ（数量 × 歩単）                 ← 累計歩工
A, b = カテゴリ別係数（下記参照）
減衰率 = MIN(1.0, INT(A × (d/b)^(-0.3) × 100 + 0.5) / 100)
歩工合計 = INT(d × 減衰率 × 100 + 0.5) / 100
```

| カテゴリ | A | b |
|----------|---|---|
| 一般機工（デフォルト） | 2.0 | 0.15 |
| エロフィン工事 | 1.2 | 0.11 |

各行への按分:
```
houkouGoukei = INT(歩工合計 × (itemD / d) × 100 + 0.5) / 100
```

### 5-4. 労務費・法定福利費
```
人工単価 = 47,500円（固定）
人工原価 = 29,400円（固定）

労務費 = INT(47,500 × 全グループ減衰後歩工合計 / 1000 + 0.5) × 1000
       ※ 手入力がある場合は手入力値を優先

法定福利費 = ROUND(労務費 × 法定福利費率 / 100)
           ※ デフォルト率 14.6%
```

### 5-5. 合計金額
```
明細合計  = Σ（全明細の金額）
値引き後  = 明細合計 − 値引き額
貴社お渡し価格 = 値引き後（①基本情報・③PDF出力で同期表示）
最終合計  = 貴社お渡し価格 + 労務費 + 法定福利費
```

## 6. Zoho CRM 連携

### 6-1. データ取得
| データ | Zoho モジュール | 取得方法 |
|--------|----------------|---------|
| 商品マスタ | Products | ZOHO.CRM.API.searchRecord |
| 工事費マスタ | CustomModule1 | ZOHO.CRM.API.searchRecord |
| 見積保存済みJSON | Quotes.field | fetchById |

### 6-2. 保存（saveToCRM）
保存先フィールド:
| Zoho フィールド | 内容 |
|----------------|------|
| Subject | 見積番号（CQR〇〇〇〇〇-改訂番号） |
| field60 | 明細合計（定価合計） |
| Description | 見積全データ（JSON文字列） |
| LinkingModule1 | 工事費サブフォーム（明細行） |

### 6-3. LinkingModule1 サブフォームフィールド
| ウィジェット | Zoho フィールド |
|-------------|----------------|
| 品名 | quoteType（1行テキスト） |
| 型番・規格 | Product_Code |
| 数量 | quantity |
| 単位 | Usage_Unit |
| 単価 | Unit_Price |
| 金額 | field10 |

### 6-4. サブフォーム重複防止
- 保存成功後、レスポンスから LinkingModule1 の行IDを取得し `state.subformRowIds` に保存
- 再保存時は `row.id` を付与してUPDATE（IDなし = INSERT）
- 削除行は `{ id: xxx, '$state': 'delete_record' }` でMarkし送信

## 10. 既知の制約・注意事項

1. **単価はマスタに依存**: ウィジェット内で単価を計算しない。Zoho マスタ（Products / CustomModule1）の値がそのまま表示される。
2. **歩単はマスタ固定値**: 計算式なし。マスタの `field_houdan`（API名要確認）を参照。
3. **減衰計算の係数**: 現在 Ca/Cb 等の歩工区分ではなく `calcCategory`（エロフィン含む文字列）で分岐。区分との対応は要確認。
4. **labor Cost 自動計算**: 手入力がある場合は手入力優先。クリアすると自動計算に戻る。
5. **PDF生成**: ブラウザの印刷機能を使用。Zoho Widget の blob URL 環境で動作。
